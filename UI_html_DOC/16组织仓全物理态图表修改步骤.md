# 16 组织仓全物理态图表修改步骤

本文记录本次 `7749e05 重构组织仓全物理态图表` 的实际修改步骤。

## 1. 确认需求口径

1. DECO 主卡片 16 组织仓保持竖向 16 根柱。
2. 数据源从 `relative_gf_percent[16]` 改为 `absolute_gf_percent[16]`。
3. `absolute_gf_percent` 允许负数，不能在算法适配层截成 0。
4. 图表 Y 轴按 `-100..120` 映射。
5. 0% 是环境水压基准线，`current_target_gf` 是动态 GF 虚线，100% 是 M 值线。
6. 负值柱从 0% 向下画；正值柱从 0% 向上画。
7. 正值未超过 GF 线为绿色，超过 GF 线部分为黄色，超过 100% 部分为红色闪烁，并在 120% 封顶。

## 2. 修改数据通路

1. 在 `src/ui/core/ui_types.h` 中把 `sensor_data_t.tissue_raw_pct[16]` 从 `uint8_t` 改为 `int16_t`。
2. 在 `src/ui/core/ui_types.h` 中新增 `sensor_data_t.tissue_target_gf_pct`，用于保存算法返回的当前 target GF 百分比。
3. 在 `src/ui/core/data.h` 中修改 `bus_set_tissue_loads()` 签名：

```c
void bus_set_tissue_loads(const int16_t tissue_raw_pct[16],
                          const uint8_t tissue_gf_pct[16],
                          float tissue_target_gf_pct);
```

4. 在 `src/ui/core/data.h` 中把 `bus_get_tissue_raw_pct()` 返回值改为 `int16_t`。
5. 在 `src/ui/core/data.h` 中新增 `bus_get_tissue_target_gf_pct()`。
6. 在 `src/ui/core/data.c` 中同步修改 `bus_set_tissue_loads()`：
   - raw 数组按 `sizeof(g_sensor_data.tissue_raw_pct)` 比较和拷贝。
   - GF 数组按 `sizeof(g_sensor_data.tissue_gf_pct)` 比较和拷贝。
   - `tissue_target_gf_pct` 参与变化判断。
   - 写入 `g_sensor_data.tissue_target_gf_pct`。
   - 保持 `DIRTY_TISSUE_TOX` 刷新链路。
7. 在 `src/ui/core/data.c` 中新增 `bus_get_tissue_target_gf_pct()`。

## 3. 修改 VM 透传

1. 在 `src/ui/core/vm/ui_vm_dashboard_types.h` 中把 `ui_vm_deco_t.tissue_raw_pct[16]` 从 `uint8_t` 改为 `int16_t`。
2. 在 `src/ui/core/vm/ui_vm_dashboard_types.h` 中新增 `ui_vm_deco_t.tissue_target_gf_pct`。
3. 在 `src/ui/core/vm/ui_vm_dashboard.c` 中，从 bus 路径读取 `bus_get_tissue_target_gf_pct()`。
4. 在 `src/ui/core/vm/ui_vm_dashboard.c` 中，从 `sensor_data_t` 路径拷贝 `sensor->tissue_target_gf_pct`。
5. 保持 `tissue_gf_pct[16]` 不变，用于旧小组件和信息概览兼容。

## 4. 修改 PC 算法适配层

1. 在 `src/algo_sim/deco_core.cpp` 中新增 `round_i16_pct(float value)`：
   - 非有限值返回 0。
   - 小于 `-32768` 时封到 `-32768`。
   - 大于 `32767` 时封到 `32767`。
   - 正负数分别按四舍五入转成 `int16_t`。
2. 在 `src/algo_sim/deco_core.cpp` 中新增 `target_gf_percent_from_core(float current_target_gf)`：
   - 无效或小于等于 0 时回退到当前 `s_gf_high_pct`。
   - `0..1` 小数域转换为百分比。
   - 已经是百分比域时直接返回。
3. 在 `sync_tissue_data()` 中把 `tissue_raw` 从 `uint8_t[]` 改为 `int16_t[]`。
4. 在 `sync_tissue_data()` 中用 `round_i16_pct(gradients.absolute_gf_percent[i])` 写入 raw。
5. 保留 `relative_gf_percent` 到 `tissue_gf[]` 的旧兼容链路。
6. 调用新的 `bus_set_tissue_loads(tissue_raw, tissue_gf, target_gf_percent_from_core(gradients.current_target_gf))`。

## 5. 修改模拟数据入口

1. 在 `src/hal_sim/sim_data.c` 中把 `s_tissue_raw[16]` 从 `uint8_t` 改为 `int16_t`。
2. 计算兼容 `tissue_gf[]` 时，只对正数 raw 计算 GF 百分比。
3. 调用新的 `bus_set_tissue_loads(s_tissue_raw, tissue_gf, (float)gf_high)`。

## 6. 修改旧小组件兼容

1. 在 `src/ui/comp/comp_view.c` 中把组织仓小组件绘制局部变量从 `uint8_t pct` 改为 `int draw_pct`。
2. `COMP_TISSUE_RAW_4012` 使用有符号 `vm->tissue_raw_pct[i]`。
3. 紧凑型 raw 小组件只在绘制层把负值按 0 处理，不改 data bus 或 VM 原始值。
4. `COMP_TISSUE_GF_4012` 继续使用 `vm->tissue_gf_pct[i]`，保持 0..120 旧概览量程。

## 7. 重构 DECO 主卡片绘图

1. 在 `src/ui/cards/card_deco.c` 中移除旧的 16 个 `s_bars[]` LVGL 子对象方案。
2. 在 `src/ui/cards/card_deco.c` 中新增单个 `s_tissue_chart` 自绘对象。
3. 删除旧 `s_mvalue_line`、`s_mvalue_label`、柱高缓存、柱色缓存和旧 M-value 线更新逻辑。
4. 新增绘制常量：

```c
#define TISSUE_DRAW_MIN_PCT      (-100)
#define TISSUE_DRAW_MAX_PCT      120
#define TISSUE_BASELINE_PCT      0
#define TISSUE_MVALUE_PCT        100
#define TISSUE_TARGET_FALLBACK   85
```

5. `any_tissue_danger()` 改为判断 `tissue_raw_pct[i] > 100`。
6. 红色危险闪烁 timer 改为只切换 `s_tissue_flash_phase` 并 invalidate 整个 `s_tissue_chart`。
7. 新增 `tissue_draw_pct_for_range()`，只做绘制范围 `-100..120` 限制。
8. 新增 `tissue_y_for_pct()`，把百分比映射到图表 Y 坐标。
9. 新增 `tissue_target_gf_draw_pct()`，把 VM 中的 target GF 转成绘制百分比。
10. 新增 `tissue_draw_line()`，用于绘制 0% 实线、动态 GF 虚线、100% 实线。
11. 新增 `tissue_draw_bar_segment()`，用于同一根柱子的分段绘制。
12. 新增 `tissue_chart_draw_cb()`：
    - 每列先画暗绿色背景。
    - 负值段从 0% 到负值向下画青绿色。
    - 正值到 GF 线之间画绿色。
    - GF 线到 100% 之间画黄色。
    - 100% 到实际值之间在闪烁相位为真时画红色。
    - 每列底部继续绘制 1..16 编号。
    - 最后绘制 0% 基准实线、动态 GF 虚线、100% M 值实线。
13. `card_deco_create()` 改为创建 `s_tissue_chart` 并绑定 `LV_EVENT_DRAW_MAIN`。
14. `card_deco_update_vm()` 不再逐个更新子对象，只在数据变化后 invalidate `s_tissue_chart`。
