# 16 组织仓全物理态图表渲染规范

本规范用于前端、UI 和测试对齐 DECO 卡片 16 组织仓图表。当前重构彻底抛弃“截断负数”的旧做法，改为“全物理态真实映射”。

## 1. 核心坐标系与三条基准横线

整个柱状图是 Y 轴范围为 `-100%..120%` 的动态画布。所有 16 根柱子保持竖向排列。

画布上必须渲染以下三条水平基准线：

### 0% 基础实线

- 物理意义：体内气压等于当前外界水压的绝对平衡点，也是所有柱子生长的唯一原点。
- UI 表现：画布中下部的绿色实线。

### 动态 GF 虚线

- 物理意义：潜水员设定的保守度安全红线。
- 数据源：实时读取 `tissue_target_gf_pct` 绘制。
- UI 表现：画布中上部的黄色虚线，会随深度和减压阶段动态上下浮动，不能画死。

### 100% 极限实线

- 物理意义：组织的绝对生理极限。
- UI 表现：画布上部的红色实线。

## 2. 柱子的双向生长与分段变色

数据源统一使用算法 `absolute_gf_percent[16]`，进入 UI data bus 后对应有符号 `tissue_raw_pct[16]`。该数据允许负数，不得在算法适配层、data bus 或 VM 层截断为 0。

所有柱子以 0% 绿色实线为地平线，向上或向下双向生长，并按到达区域分段变色。

### 场景 A：欠饱和吸气阶段

- 条件：`value < 0%`。
- 生长方向：从 0% 绿线开始，向下反向生长。
- UI 表现：纯青绿色或暗绿色，表示当前深度下外界水压较大，组织处于欠饱和吸气阶段。

### 场景 B：正常过饱和排气阶段

- 条件：`0% < value <= tissue_target_gf_pct`。
- 生长方向：从 0% 绿线开始，向上正向生长。
- UI 表现：亮绿色，表示处于安全的减压排气范围。

### 场景 C：违规越界

- 条件：`tissue_target_gf_pct < value <= 100%`。
- UI 表现：柱子向上冲破动态 GF 虚线的那一截必须渲染为黄色。

### 场景 D：极度危险

- 条件：`value > 100%`。
- UI 表现：柱子冲破 100% 红线的那一截必须渲染为红色且闪烁。
- 限幅保护：无论后台数据多大，前端必须在 120% 处做绘制封顶，防止柱子穿过 UI 卡片边框。

## 3. 测试与联调验证点

### 下潜初中期

必须看到所有柱子都在 0% 绿线下方，并从 0% 线向下生长。

### 上升减压期

柱子应从下方缩回，穿过 0% 绿线后开始向上生长。

### 动态 GF 虚线

减压停留期间，黄色 GF 虚线应随算法反馈的 `tissue_target_gf_pct` 动态上下位移，不能固定在某个常量高度。

### 分段颜色

构造或接入数据时，需要覆盖以下区间：

- `value < 0%`：只出现向下的青绿色或暗绿色段。
- `0% < value <= tissue_target_gf_pct`：只出现向上的亮绿色段。
- `tissue_target_gf_pct < value <= 100%`：超过 GF 虚线的部分为黄色。
- `value > 100%`：超过 100% 红线的部分为红色闪烁，并在 120% 封顶。

## 4. 兼容边界

- DECO 主卡片必须使用 `absolute_gf_percent[16]` / `tissue_raw_pct[16]`。
- `relative_gf_percent[16]` / `tissue_gf_pct[16]` 只保留给旧小组件和信息概览兼容。
- 只有绘制边界允许做 `-100..120` 限幅，业务数据本身不得被改写为这个范围。
