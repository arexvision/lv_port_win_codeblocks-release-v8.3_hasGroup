# 16 组织仓归一化视图算法沟通版

本文是给算法工程师的沟通收口版。此前我们提出“算法输出组织压力物理真值，UI 负责 0..1000 映射”的方案，AREX Deco Core `0.0.23` 已经采纳。

## 已采纳结论

- UI 不再要求算法直接输出屏幕坐标或柱长。
- 算法通过 `arex_deco_calculate_tissue_pressures()` 输出当前压力域物理真值。
- UI/PC 适配层只负责把物理真值映射成 `tissue_bar_permille[16]`、`tissue_pi_permille` 和小组件显示。
- 旧的 `arex_deco_calculate_tissue_gradients()` / `ArexDecoTissueGradientMetrics` 已废弃，UI 不再用 `absolute_gf_percent` 反推 M 值。

## 当前消费字段

`ArexDecoTissuePressureMetrics` 当前已满足 UI 主图和小组件需要：

- `ambient_pressure_bar`
- `inspired_n2_bar`
- `inspired_he_bar`
- `tissue_n2_bar[16]`
- `tissue_he_bar[16]`
- `tissue_m_value_bar[16]`
- `tissue_m_gf_bar[16]`
- `current_gf_target`

## UI 映射口径

- 横向组织图范围固定为 `0..1000`。
- `400` 表示 `P_tissue == P_amb`。
- `900` 当前 RAW 主图表示 `P_tissue == M_i`。
- `1000` 是 UI 绘制封顶。
- `tissue_pi_permille = ((inspired_n2_bar + inspired_he_bar) / ambient_pressure_bar) * 400`。
- `tissue_bar_permille[i]` 使用 `P_tissue_i = tissue_n2_bar[i] + tissue_he_bar[i]`。
- Leading Tissue 小组件同样按 `PN2 + PHe` 和 `tissue_m_value_bar[i]` 计算最大 GF。

## 仍需后续确认

1. GF 模式主图是否应把 `900` 红线从 `tissue_m_value_bar[i]` 切到 `tissue_m_gf_bar[i]`。
2. 如果算法后续愿意直接输出 leading summary，建议字段为 `leading_compartment`、`leading_gf_percent`、`leading_tissue_inert_bar`、`leading_m_value_bar`、`leading_m_gf_bar`。
3. 固件侧接入时必须同样调用 `arex_deco_calculate_tissue_pressures()`，不要恢复 UI 侧反推公式。
